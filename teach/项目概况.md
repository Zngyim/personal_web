# 项目概况

这是一份给项目维护者看的总览文档，用来快速理解这个网站“由哪些部分组成”“每个部分负责什么”“哪些是外观设置，哪些是功能逻辑”。

当前项目是一个基于 `Astro` 的静态网站，已经被改造成一个偏学术风格的个人网站，主要包含：

- 首页 `Home`
- 博客 `Blog`
- 研究页 `Research`
- 项目页 `Project`

---

## 1. 项目整体结构

项目里最重要的目录和文件如下：

- `src/pages/`
  页面入口。这里的文件基本直接对应网站路由。
- `src/layouts/`
  页面外壳。负责统一的头部、页脚、整体 HTML 结构。
- `src/components/`
  可复用组件。比如导航栏、页脚、博客条目、文章页等。
- `src/data/post/`
  博客文章内容来源，放 `.md` 或 `.mdx` 文件。
- `src/data/projects.ts`
  项目页数据来源，管理每个项目的标题、简介、缩略图等信息。
- `src/content/config.ts`
  文章内容集合的 schema，定义博客文章允许有哪些 frontmatter 字段。
- `src/utils/`
  工具函数，比如博客加载、路由 permalink 处理、日期格式化等。
- `src/config.yaml`
  网站全局配置，比如站点名、博客路径、语言、SEO。
- `src/navigation.ts`
  顶部导航和页脚导航的数据来源。
- `astro.config.ts`
  Astro 构建配置，比如 Markdown、MDX、LaTeX、图片和插件。

---

## 2. 页面层分别做什么

### `src/pages/index.astro`

网站首页。

作用：

- 显示网站简介
- 显示主题方向
- 显示最近几篇博客文章
- 引导用户进入 `Blog` 或 `Research`

### `src/pages/research.astro`

研究页。

作用：

- 展示研究兴趣
- 展示长期研究方向
- 展示正在进行中的主题
- 作为比博客更稳定的研究档案页

### `src/pages/projects.astro`

项目列表页。

作用：

- 汇总所有项目
- 显示项目缩略图
- 显示项目标题
- 显示项目简介
- 跳转到单个项目详情页

### `src/pages/projects/control-system-animation.astro`

单个项目详情页。

作用：

- 展示“自动控制原理框图”项目
- 读取并嵌入独立的 HTML 项目页面
- 作为 `Project` 下某个项目的落地页

### `src/pages/[...blog]/[...page].astro`

博客列表页。

作用：

- 展示所有已发布博客
- 支持分页
- 作为 `Blog` 栏目的主入口

### `src/pages/[...blog]/index.astro`

博客文章详情页路由入口。

作用：

- 把单篇文章渲染为完整文章页面
- 调用文章详情组件进行展示

### `src/pages/[...blog]/[category]/[...page].astro`

博客分类页。

作用：

- 按分类显示文章
- 支持分页

### `src/pages/[...blog]/[tag]/[...page].astro`

博客标签页。

作用：

- 按标签显示文章
- 支持分页

## 3. 哪些部分属于“外观设置”

这里的“外观设置”指页面长什么样、字体颜色是什么、间距如何、导航和页脚怎么排版。

### `src/components/CustomStyles.astro`

负责全局视觉变量。

主要控制：

- 字体族
- 主题主色
- 文本颜色
- 背景颜色
- 深色模式颜色变量

### `src/assets/styles/tailwind.css`

负责全局样式补充。

主要控制：

- 按钮样式
- Header 滚动样式
- 菜单展开样式
- KaTeX 数学公式显示细节
- 一些全局 utility class

### `src/components/widgets/Header.astro`

负责顶部导航栏的外观与结构。

主要控制：

- Logo 区域
- 顶部导航排版
- 移动端菜单按钮
- RSS / 主题按钮显示方式

### `src/components/widgets/Footer.astro`

负责页脚外观与结构。

主要控制：

- 页脚整体排版
- 页脚链接排列方式
- 页脚说明文字

### `src/components/Logo.astro`

负责左上角站点名称显示方式。

### `src/layouts/Layout.astro`

负责全站 HTML 外壳。

主要控制：

- 全局 CSS 引入
- KaTeX CSS 引入
- 页面 `<head>` 内容挂载
- 页面 `<body>` 外层结构

### `src/layouts/PageLayout.astro`

负责普通页面统一包裹：

- 页头
- 主内容区域
- 页脚

## 4. 哪些部分属于“功能逻辑”

这里的“功能逻辑”指页面为什么能显示内容、文章为什么能自动被收录、项目为什么能跳转、公式为什么能渲染等。

### `src/navigation.ts`

功能：

- 定义顶部导航数据
- 定义页脚导航数据
- 控制网站有哪些主栏目

### `src/config.yaml`

功能：

- 定义站点基础信息
- 定义博客主路径
- 定义分类路径
- 定义标签路径
- 定义文章分页数量
- 控制是否开启博客等功能

### `src/content/config.ts`

功能：

- 定义博客文章 frontmatter 的合法字段
- 约束 `publishDate`、`title`、`tags`、`category`、`draft` 等字段格式
- 决定哪些 `.md/.mdx` 文件能被当作文章读取

### `src/utils/blog.ts`

功能：

- 加载所有博客文章
- 对文章进行标准化处理
- 根据 `publishDate` 排序
- 过滤 `draft: true` 的文章
- 生成博客列表分页
- 生成单篇文章路径
- 生成分类页路径
- 生成标签页路径
- 获取最新文章

### `src/utils/permalinks.ts`

功能：

- 统一生成页面链接
- 统一生成博客链接
- 统一生成分类和标签链接
- 处理站点 base path

### `src/utils/utils.ts`

功能：

- 格式化日期
- 一些通用字符串与数值工具

### `astro.config.ts`

功能：

- 配置 Astro 构建行为
- 配置 Markdown / MDX 支持
- 配置 LaTeX 公式支持
- 配置 sitemap
- 配置 Tailwind
- 配置压缩
- 配置图标

### `src/data/post/`

功能：

- 作为博客文章数据源
- 每个 `.md` / `.mdx` 文件对应一篇博客文章

### `src/data/projects.ts`

功能：

- 作为项目列表数据源
- 每个项目包含 slug、标题、简介、缩略图
- 项目列表页和项目详情页都可以复用这里的数据

## 6. 目前最常需要修改的地方

如果后续只是日常维护网站，最常改的是这些：

- `src/data/post/`
  新增或修改博客文章
- `src/data/projects.ts`
  新增或修改项目条目
- `src/pages/index.astro`
  修改首页文案
- `src/pages/research.astro`
  修改研究页内容
- `src/navigation.ts`
  修改顶部导航和页脚导航
- `src/config.yaml`
  修改站点名字、域名、博客路径等

## 7. 一句话理解这个项目

可以把这个项目理解成：

“一个用 Astro 搭建的静态个人学术网站，其中 `pages` 决定页面，`data/post` 决定博客内容，`data/projects.ts` 决定项目内容，`layouts/components/styles` 决定外观，`utils/config` 决定功能逻辑与站点行为。”
